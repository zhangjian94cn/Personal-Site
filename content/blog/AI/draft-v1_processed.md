## 🛠 怎么搭？5 步

### Step 1：在飞书创建 3 个机器人

登录 [飞书开放平台](https://open.feishu.cn)，创建 3 个自建应用，分别对应 CEO、产品、研发。

每个应用就做 3 件事：启用机器人功能、配上事件订阅（长连接模式）、发布上线。

> 💡 和之前给龙虾创建应用一模一样，只不过这次建 3 个。给每个机器人配上不同的名字和头像——这样它们在群里才像 3 个不同的人。

### Step 2：创建 3 个 Agent

在服务器上创建 3 个 OpenClaw Agent，各自有独立的"大脑"和"人格"：

```bash
openclaw agents add ceo --workspace ~/.openclaw/workspace-ceo
openclaw agents add product --workspace ~/.openclaw/workspace-product
openclaw agents add dev --workspace ~/.openclaw/workspace-dev
```

每个 Agent 都有自己独立的工作目录，你可以给它们写不同的 SOUL.md——CEO 的沉稳、产品的敏锐、研发的务实，全都靠这里定义。

### Step 3：把机器人和 Agent 绑在一起

这步是核心——告诉 OpenClaw：哪个飞书机器人对应哪个 Agent。

在配置文件的 `accounts` 里填上 3 个飞书应用的 appId，然后在 `bindings` 里做一对一绑定：

| 飞书机器人 | 绑定的 Agent | 角色 |
|-----------|------------|------|
| 若言 | ceo | CEO |
| 观澜 | product | 产品经理 |
| 知行 | dev | 研发工程师 |

就是这么简单——3 个机器人，3 个 Agent，一一对应。

### Step 4：验证

```bash
openclaw channels status
```

看到 3 个 Feishu account 都是 `running`，搞定。

### Step 5：配对

最后一步：分别和 3 个机器人各说一句话，触发配对流程，逐个 approve 就行：

```bash
openclaw pairing approve feishu <配对码>
```

## 💣 4 个坑，我替你踩了

搭建过程简单，但有 4 个隐藏的坑。不提前知道，能让你折腾半天。

**坑 1：配置里有个地方要写两遍**

顶层配置和 accounts 配置里都要填 default 的 appId。少写一个，直接报错 `not configured`。别问我怎么知道的。

**坑 2：同一个人，在 3 个机器人眼里是不同的人**

飞书的规则是——不同应用看到同一用户的 ID 是不一样的。这个叫 `open_id 隔离`。如果你在代码里用 ID 做什么关联，要注意这一点。

**坑 3：配对要做 3 次**

配了若言不代表观澜和知行也能用。每个机器人都要单独配对。3 个 bot × N 个用户 = 3N 次 approve。

**坑 4：模型没余额，全军覆没**

如果某个 Agent 指定的模型提供方余额用光了，不只是这一个 Agent 挂——同模型的其他 Agent 也会一起返回 403。报错还不告诉你是余额问题，非常迷惑。
