31.8k Star。5000 Fork。301 人 Watch。142 次 Commit。

这组数据属于一个叫 agency-agents 的 GitHub 项目。

它不是框架，不是模型，不是又一个 AI 工具。

它是一家公司——一个人的公司——有 120 个 AI 员工。

文末给大家准备了完整的 Agent 清单整理 + 快速上手 cheatsheet，别忘了领。

## 从 Reddit 一个帖子到 3 万 Star

故事的起点很简单。

Michael Sitarzewski，一个写了 30 年代码的老程序员，Techstars 校友，连续创业者。他在 Reddit 的 r/ClaudeAI 板块上发了一个帖子，说自己造了一批 Claude Code 的 Agent 人设，问有没有人感兴趣。

12 小时内，50 多个人举手说要。

然后事情就失控了。

两周之内，Star 数从零直线拉到 3 万。你看一眼 Star History 的曲线就知道什么叫"垂直起飞"——曲线根本不是增长，是发射。

![Star History 趋势图](https://assets.zhangjian94cn.top/images/blog/agency-agents_processed/agency-agents-star-history.png)

这不是一个精心策划的开源营销案例。这是一个人在解决自己的问题，恰好引爆了整个社区的需求。

## 这 120 个"员工"到底是什么

你可能在想：不就是一堆 Prompt 吗？

不完全是。

每个 Agent 都有完整的"入职档案"：身份设定、性格特征、核心使命、工作流程、交付标准、成功指标，甚至有沟通风格。它不是那种"你是一个资深前端工程师"的一句话 Prompt——它是一套完整的人格系统。

![GitHub 仓库主页](https://assets.zhangjian94cn.top/images/blog/agency-agents_processed/agency-agents-github-homepage.png)

这 120 个 Agent 分布在 12 个部门：

**💻 工程部**：前端开发、后端架构师、移动端、AI 工程师、DevOps、安全工程师……光工程部就有 15 个人。

**🎨 设计部**：UI 设计师、UX 研究员、品牌守护者，还有一个叫"Whimsy Injector"的角色——专门往产品里注入趣味性。它会说："让我加个庆祝动画，把用户完成任务后的焦虑降低 40%。"

**📢 营销部**：增长黑客、内容创作者、Twitter 运营、TikTok 策略师、Reddit 社区运营……

**💰 付费媒体部**：PPC 投放、搜索词分析师、广告审计、程序化购买。

**💼 销售部**：外拓策略师、发现教练、交易策略师、销售工程师。

**📊 产品部**：Sprint 排序、趋势研究、反馈综合、行为助推引擎。

**🎮 游戏开发部**：还分了 Unity、Unreal、Godot、Roblox 四条线。

想象一下，你一个人坐在电脑前，打开 Claude Code，说一句"帮我搭个创业 MVP"。前端开发开始写 React，后端架构师设计 API，增长黑客规划获客策略，快速原型师做迭代，Reality Checker 做上线前检查——五个 Agent 同时开工。

这才是"一个人的全栈公司"。

还有一个微信小程序开发者 Agent，写在 engineering 目录下，记住了各种审核被拒的原因和性能优化模式。

一个 30 年经验的美国开发者，认认真真地为中国互联网做了 7 个专属 Agent。

我不确定他是自己研究的，还是有中国社区贡献者帮忙——但无论如何，这份覆盖面本身就说明了一件事：AI Agent 的"人设工程"，正在成为一个全球性的基础设施需求。

## 三分钟上手

上手极其简单。

如果你用 Claude Code：

```bash
cp -r agency-agents/* ~/.claude/agents/
```

复制完，对话里说"启用 Frontend Developer 模式"就行了。

如果你用 Cursor、Aider、Windsurf 或者 Gemini CLI：

```bash
./scripts/convert.sh    # 生成各工具的适配文件
./scripts/install.sh    # 自动检测并安装
```

MIT 协议，随便用，随便改。

已经有中文社区做了翻译版本。也有人基于它搞了 OpenClaw 的 Agent 集合。

## 有意思的地方在于"模式"

这个项目真正有价值的，不是那 120 个具体的 Agent——那些你可以自己写。

有价值的是它证明了一个模式：

**第一**，AI Agent 不应该是通用的。"你是一个资深工程师"这种 Prompt 效果远不如一个完整的人格系统。深度专业化 > 通用模板。

**第二**，Agent 需要"性格"。不是为了好玩，而是因为性格决定了沟通风格和决策偏好。一个严谨的安全工程师和一个激进的增长黑客，面对同一个问题会给出完全不同的建议。

**第三**，Agent 要有"交付标准"。成功指标、工作流程、输出格式——这些约束让 Agent 的输出从"差不多"变成"可以直接用"。

**第四**，一个人可以管理的 Agent 数量正在指数增长。120 个还只是开始。

想想看，去年这个时候我们还在讨论 Prompt Engineering。现在的问题已经变成了：怎么给 AI 组建一个公司？

韩信点兵，多多益善——不是因为他自己能打，而是他有一套体系，给他多少兵都能管。

这个项目就是那套体系的雏形。
