# OpenClaw 到底是什么？一个被吹上天的项目，我决定自己试试

> 过年在家刷手机，公众号被一个叫 OpenClaw 的东西刷屏了。有人叫它"草根造神运动"，有人说"小白用户活不过一周"。我看了一圈，越看越糊涂。后来想，算了，自己装一个吧。

## 一、刷屏

OpenClaw 这个名字，我其实关注有一阵子了。

但真正想自己动手试一试，是过年那几天。放假在家刷手机，公众号又开始了一轮密集推送。不是某一个号在写，而是好几个完全不同的号——做自媒体的、做投资分析的、做技术教程的——同时在聊它。

闲着也是闲着，我把收藏夹里攒的那些文章翻出来认真看了一遍。

第一篇标题里带着双重感叹号，说这是「2026 年引爆 Agent 狂潮的开源项目」，旧金山几千人头戴龙虾帽搞线下聚会，Mac Mini 因为它卖断了货。文章结尾的建议是"暴力试错"，谁先跑通谁就赢。

第二篇画风完全不同。一个真实用户拉了一张清单，列了自己让 OpenClaw 干的活：搬运公众号文章到个人主页、识别打赏截图更新赞助记录、定时抓取监管信息、监控邮箱过滤消息。听着挺实在的。但往下翻，他花了大半篇幅写「劝退指南」——工作区三天不清理就面目全非，配置文件经常被 AI 自己改崩，他得开着另一个 AI 在旁边随时准备修。最后一句话我记得很清楚：**"如果你平时下班甚至会感到无聊，那你大概率是没有这个需求的。"**

第三篇是商业分析，用「挑榴莲」来类比产品定位。结论是 OpenClaw 看起来像终极形态，但实际上大家的用法出人意料的类似：定期推送、提醒清单、搜索代替。一个刚好够用的中间层。

同一个东西，三篇文章，三部电影。

## 二、拆开看

过了两天，又找了几篇来读。这次不是为了被说服，是想搞清楚：这些作者到底在说同一个东西吗？

慢慢地，我把这些声音分成了四类。

**鼓吹派**用的词最大。"核心突破"、"执行权"、"平民英雄"、"造山运动"。他们会列数字：旧金山活动挤进数千人，有人推出 119 美元的一键部署服务三天赚了 1.7 万美元，垂直技能包已经超过三万个。单看每一条可能都是事实，但放在一起读，很像在看一份投资路演 PPT。

**实战派**写的东西最落地。搬运文章、识别截图、拉取 Issue、监控邮箱——全是具体的活儿。但每一项都是作者自己手搓的 Skill。那位用户说得很明白：刚装好的 OpenClaw 几乎什么都不会，搜索、浏览器自动化这些基础能力都得自己配。安装只是一切的开始。

然后他花了更多篇幅写它的问题。工作区里，AI 干活时产生的中间文件如果不清理，几天就一团乱。它可以修改自己的配置文件，但经常改崩，得有人在旁边守着。Token 消耗惊人，每月 200 元起步还不是用的最好的模型。他甚至建议不要在日常用的电脑上部署——得专门有一台闲置的机器让它折腾。

**分析派**退后一步看全局。有一篇用"挑榴莲"来打比方：太保守（把需求拆太细）和太激进（押注终极形态）都不行，只有中间那一层才能吃。OpenClaw 看起来很激进——AI 操作电脑做一切——但你去看真实用户在干什么，来来去去就那几样。它是个中间层。不优雅，但刚好够用。

**劝退派**最冷静，说的话也最扎。"装完什么都不会"，"配置文件被 AI 自己改崩"，"对小白来说活不过一周"。这些不是故意黑，就是用过的人的大实话。

有意思的是，这四种说法并不矛盾。它们描述的确实是同一个东西——只是每个人站的位置不同，看到的面不同。

## 三、所以它到底是什么

看了这么多别人写的，我试着自己说一遍。

### 来历

OpenClaw 最初叫 Clawbot，是奥地利开发者 Peter Steinberger 在 2026 年 1 月发布的。因为 Anthropic 的法律压力改名 Moltbot，最终定名 OpenClaw。一月初在旧金山办了一场叫 ClawCo 的社区活动，几天内挤进数千人，大家头戴红色龙虾帽合影。从那以后，这个项目就开始在全球开发者圈子里快速传播。

名字换了三遍，但做的事一直没变。

### 一句话解释

**OpenClaw 是一个自托管的 AI Agent 网关。**

拆开来讲：

**自托管。** 跑在你自己的机器上——笔记本、Mac Mini、服务器都行。数据不经过第三方，API Key 自己管。

**AI Agent 网关。** 它自己不是 AI，不做推理，不生成文字。它是个中间人：你在 WhatsApp 或 Telegram 里发一条消息，OpenClaw 接到后转给 AI 模型（Claude、GPT、Kimi，随你配），再把回复转发回来。

**多通道。** 一个 Gateway 进程同时接 WhatsApp（通过 Baileys 库实现 WhatsApp Web 协议）、Telegram（通过 grammY 框架）、Discord、iMessage、Mattermost。你不需要为每个平台单独跑一个服务。

这就是它和 ChatGPT App、Lobe Chat 那些工具最大的区别：**你不需要打开一个新的界面**。聊天入口就是你已经在用的通讯工具。

### 用一个类比来说

OpenClaw 相当于给 Claude 装了一部手机。

你的 AI 模型本来只能在网页上等你打开浏览器去找它。现在 OpenClaw 给它注册了 WhatsApp 号码、Telegram 账号，让它能随时接你的消息。它还可以同时注册多个号、甚至有多个"人格"分别回复不同的人——这就是官方说的 multi-agent routing。

但就像新手机开机之后你得自己装 App 一样，OpenClaw 装好之后也得你自己写 Skill、配 Channel、喂设定，才能干有用的事。那些实战派用户列出的花样——搬运文章、识别截图、监控邮箱——全是他们自己手搓的。

### 技术栈一览

从官方文档里整理一下，给自己留个底：

- **安装方式**：`npm install -g openclaw@latest`，然后 `openclaw onboard` 走引导流程（也可以 Docker 部署）
- **核心进程**：Gateway，一个 Node.js 常驻服务，默认监听 `18789` 端口
- **配置文件**：`~/.openclaw/openclaw.json`，JSON 格式，手动改或通过 Dashboard 改
- **Dashboard**：浏览器打开 `http://127.0.0.1:18789/`，可以看会话、改配置、管 Channel
- **模型接入**：支持 Anthropic、OpenAI 等，通过 OAuth 或 API Key 配置
- **能力扩展**：Skills 系统——本质上是你写给 AI 的"工具箱"，每个 Skill 一个目录，里面放脚本和说明

### 它不是什么

有几个概念我一开始搞混了，顺便澄清：

**不是大模型。** 你得自己提供 API Key，OpenClaw 只管转发。

**不是聊天 App。** 手机上不需要装一个叫"OpenClaw"的 App 来聊天。（有 iOS/Android App，但那是做 Node 扩展的——拍照、定位、Canvas 画布——不是聊天入口。）

**不是开箱即用的。** 这一点所有用过的人都在反复说。装好之后它连搜索都不会，Skill 得自己写，Channel 得自己配。官方文档也很坦诚：如果你什么都不改，它默认走内置的 Pi 模型做 per-sender session——能聊天，但也仅此而已。

## 四、我想搞清楚的事

说了这么多别人的话，是时候自己动手了。

我的做法很简单：部署起来，接上聊天软件，用一段时间，看看到底怎样。

有几个问题特别想搞清楚：

- **部署有多难？** 我不是 Node.js 选手，平时主要写 Python。对我这种人，部署过程有多少坑？
- **日常体验怎么样？** 手机上发一条消息、AI 秒回——听起来很爽，但延迟多少？图片能不能发？群聊里会不会乱插嘴？
- **那些高级功能有没有用？** 多 Agent、Skills、定时任务——对我这种普通用户，哪些是真需求，哪些是花架子？

还有一个问题，我觉得比上面这些都重要：

**一个月之后，我还会继续用吗？**

很多工具装好的那一刻让人兴奋，两周后就安静地躺在 Docker 里吃灰。OpenClaw 会不会也走这条路，我不知道。用了才知道。

## 写在最后

这篇文章没有结论。因为我还没开始用。

写这个引子，是因为读那些文章的过程中，我发现一个有意思的现象：大多数人对 OpenClaw 的印象，要么来自鼓吹，要么来自劝退。两边都有道理，但拼不出一个完整的画面。

我没法判断谁对谁错。能做的就是自己装一个，用一段时间，把过程写下来。踩坑就说踩坑，好用就说好用。
